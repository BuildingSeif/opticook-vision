Section 02

Les 3 regles fondamentales du produit

Vous avez defini trois regles non negociables. Elles deviennent nos contraintes d'architecture. Chaque decision technique sera evaluee contre ces trois regles.

01
Accessibilite immediate, zero friction
« OptiCook doit exister avant meme que l'utilisateur ait decide de l'utiliser. »
Implementation technique

Mobile : Share Extension native (iOS + Android). L'utilisateur voit une recette sur Instagram, TikTok, un blog : il appuie sur "Partager" et OptiCook apparait dans la liste. Un tap = recette capturee.

Desktop : Extension navigateur (Chrome, Safari, Firefox). Un bouton flottant sur toute page contenant une recette. Detection automatique du contenu culinaire via analyse semantique.

Zero inscription : La premiere capture fonctionne sans compte. Le compte est propose apres la 3e capture, quand l'utilisateur a deja de la valeur stockee.

02
Capture universelle, absolument tout
« Si tu peux le voir, OptiCook peut le capturer. »
Implementation technique

10 sources de capture :

1. URL : Extraction structuree (schema.org, microdata, heuristique AI) depuis tout site web
2. Reseaux sociaux : Instagram, TikTok, YouTube, Pinterest, Facebook via deep link + API
3. Video : Extraction audio (Deepgram STT) + analyse visuelle (AI Vision avancee) des videos YouTube et TV
4. Photo : OCR + AI Vision pour livres, magazines, dos de paquet, menus restaurant
5. PDF : Extraction structuree de documents cuisines (livres numeriques, fiches PDF)
6. Manuscrit : Reconnaissance d'ecriture manuscrite (carnet de grand-mere, notes griffonnees)
7. Dictee vocale : L'utilisateur dicte une recette, l'AI structure automatiquement
8. Menu restaurant : Scan de carte, extraction des plats, creation de fiche restaurant
9. Plat a table : Photo d'un plat servi = identification + recette "inspiree par"
10. Partage entrant : Un ami envoie un lien = capture en un tap depuis n'importe quelle app de messagerie

03
Instantaneite totale, "Immortelle"
« Tu vois ➔ tu captures ➔ OptiCook s'occupe du reste. »
Implementation technique

Capture en arriere-plan : L'utilisateur partage un lien et retourne immediatement a ce qu'il faisait. OptiCook traite en arriere-plan (extraction AI, normalisation, enrichissement nutritionnel, traduction) et envoie une notification quand la recette est prete dans son Book.

Temps cible : <3 secondes entre le tap "Partager" et le retour a l'app d'origine. Traitement complet en <30 secondes en arriere-plan.

Zero configuration : Pas de compte requis. Pas de choix de langue initiale obligatoire (detection automatique). Pas de tutoriel force. L'app fonctionne des le premier contact.

Angles d'innovation, votre vision, notre execution

1. Traduction culinaire contextuelle

« Les apps actuelles traduisent litteralement, sans comprendre le contexte culinaire, la culture, le geste, l'intention. »

Nicolas Bertin, 30 mars 2026
Notre approche

Un moteur de traduction culinaire alimente par un modele AI entraine sur les equivalences culturelles. "Cream" dans une recette britannique = creme fraiche epaisse, dans une recette americaine = heavy cream, dans une recette indienne = malai. Le systeme adapte automatiquement les quantites, les ingredients equivalents locaux et les techniques selon le pays de l'utilisateur.

2. Multilinguisme reel

Couverture jour 1

Phase 1 : Francais + Anglais + Espagnol
Phase 2 : Arabe (RTL complet), Portugais, Allemand, Italien
Phase 3 : Hindi, Chinois simplifie, Japonais, Coreen
L'architecture i18n est native des le premier jour. Chaque ecran supporte RTL, chaque typographie est testee en CJK. Ce n'est pas un ajout : c'est dans l'ADN du code.

3. Desktop first-class citizen

Extension navigateur

Un bouton flottant apparait sur toute page contenant une recette. Detection automatique via schema.org + heuristique AI. Un clic = recette sauvegardee dans le Book, synchronisee en temps reel avec le mobile. Video YouTube capturee sur PC, exploitee en cuisine sur le telephone.
